## 简介
本文件面向需要在现有系统基础上集成第三方服务（尤其是外部 API）的团队，围绕以下主题提供系统化说明：
- 如何接入外部 API（以 Brave Search 和阿里云 DashScope 为例），包括 API 密钥管理、认证机制与请求限流处理
- 云函数集成（阿里云 FC3）、CDN/域名配置与部署
- 监控系统对接建议
- 支付网关、社交媒体分享与分析工具的集成思路
- 数据同步策略、缓存机制与故障转移方案
- 安全考虑、性能优化与成本控制建议

本项目当前已实现：
- 使用 Brave Search 提供的新闻检索能力，并在无有效密钥或失败时自动回退到爬虫与模拟数据
- 通过阿里云 DashScope 提供的通义千问模型进行文本生成与翻译，支持商品文案生成与营销文案本地化
- 通过 Next.js API 路由统一聚合 API 与本地爬取的数据
- 通过云函数（FC3）与平台（Vercel）两种方式部署
- 通过环境变量注入 API 密钥，支持本地与云端配置
- 新增文件上传与存储功能，支持图片与视频的本地存储与访问

## 项目结构
项目采用 Next.js 应用结构，第三方集成相关的关键位置如下：
- API 层：app/api/news/route.ts 聚合外部 API 与本地爬虫数据；app/api/ai-lab/* 提供 AI 实验室相关 API
- 外部 API 封装：lib/brave-search.ts 对 Brave Search 的封装与回退逻辑；lib/aliyun/dashscope.ts 对阿里云 DashScope 的封装
- 文件存储：lib/aliyun/storage.ts 提供本地文件上传与存储功能
- 爬虫层：lib/news-scraper.ts 抓取指定站点内容
- 模拟数据：lib/mock-data.ts 用于无密钥或异常时的回退
- 分类配置：lib/news-categories.ts 提供关键词与分类映射
- 视频任务管理：lib/video-tasks.ts 管理视频生成任务状态
- 前端页面：app/ai-lab/* 提供 AI 实验室交互界面
- 部署与运行时：
  - 云函数适配器：fc-handler.js 适配 FC3 HTTP 触发器
  - 阿里云资源定义：s.yaml
  - Vercel 平台配置：vercel.json
  - Next 构建输出：next.config.mjs

## 核心组件
- 外部 API 封装（Brave Search）
  - 负责构造查询参数、设置认证头、发起请求、解析结果并提供回退逻辑
  - 当新闻接口不可用时自动回退到网页搜索接口
- 阿里云 DashScope 封装
  - 基于 OpenAI SDK 封装通义千问模型，提供文本生成与翻译能力
  - 支持商品详情简介生成与营销文案本地化
- 文件存储服务
  - 提供本地文件上传、类型校验、大小限制与安全命名
  - 支持图片与视频两类文件的存储与访问
- 视频任务管理
  - 内存级任务状态管理，支持任务创建、进度跟踪与结果存储
  - 生产环境需替换为持久化存储（Redis/数据库）
- 爬虫模块
  - 基于 Cheerio 解析 HTML，按分类抓取指定站点的新闻条目
- API 路由聚合
  - 同时拉取 API 与爬虫数据，合并去重，支持关键词过滤与分类关键词拼接
  - 在无有效密钥或外部 API 异常时，回退到模拟数据与爬虫数据
- 云函数适配器
  - 将 FC3 HTTP 触发器请求转发给本地 Next.js 服务，处理启动等待与错误响应
- 部署配置
  - Next.js 输出为独立包，便于在 FC3 与 Vercel 上部署
  - 通过环境变量注入 API 密钥，支持本地与平台托管

## 详细组件分析

### 外部 API 封装（Brave Search）
- 认证机制
  - 通过请求头携带订阅令牌进行认证
  - 从环境变量读取密钥，若未配置则抛出错误
- 请求与回退
  - 优先尝试新闻搜索接口；当响应非 OK 时，自动回退到网页搜索接口
  - 返回标准化的新闻项结构，包含标题、描述、链接、来源、发布时间等
- 限流与稳定性
  - 项目未内置显式的速率限制或重试策略
  - 可通过外部服务配置与平台限流策略缓解

```mermaid

CallAPI --> RespOK{"响应是否成功？"}

Return --> End
```

## 性能考量
- 并发与去重
  - API 路由已采用并发策略获取 API 与爬虫数据；建议在爬虫模块中引入并发抓取以进一步缩短延迟
  - AI 实验室 API 中的文件上传采用串行处理，可考虑批量处理优化
- 缓存策略
  - 建议对 API 响应与爬虫结果进行短期缓存（如 Redis 或 CDN），并为不同分类设置独立 TTL
  - 对高频查询（如首页）可预热缓存
  - DashScope 文本生成结果可考虑短期缓存以降低成本
- 限流与熔断
  - 在外部 API 层引入指数退避与熔断（如连续失败超过阈值暂停请求）
  - 对 FC3 与平台触发器设置合理的超时与并发上限
  - DashScope API 需要遵守阿里云的配额限制
- 资源优化
  - 使用独立包输出与静态图片处理减少冷启动与构建体积
  - 对响应启用压缩（Gzip/Brotli）与 CDN 边缘缓存
  - 文件存储建议配合 CDN 加速静态资源访问

## 故障排查指南
- API 密钥问题
  - 症状：抛出"密钥未配置"错误或外部 API 响应非 OK
  - 处理：检查环境变量配置；确认订阅令牌有效且未过期
  - DashScope：检查 DASHSCOPE_API_KEY 是否正确设置
- 外部站点不可用
  - 症状：爬虫抓取失败或返回空结果
  - 处理：查看日志定位失败站点；确保网络可达与 UA/Headers 合法
- 云函数启动慢
  - 症状：首次请求返回"服务启动中"
  - 处理：等待服务就绪；检查适配器中的等待循环与超时设置
- 平台部署差异
  - 症状：本地与平台行为不一致
  - 处理：核对平台变量与本地 .env.local 的差异；确认构建与运行时版本一致
- 文件上传失败
  - 症状：返回"文件过大"或"不支持的文件类型"
  - 处理：检查文件类型与大小限制；确认上传表单格式正确
- AI 生成异常
  - 症状：DashScope 调用失败或返回错误
  - 处理：检查 API 密钥与配额；查看网络连接状态
